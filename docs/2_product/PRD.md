# Welldy PRD (Product Requirements Document)

---

## 제품 개요

- **제품명**: Welldy (웰디)
- **이전 제품명**: CoachMate, WellCoach, ExWell, Wellbud, X Coach, ALLinFIT
- **한 줄 정의**: AI 운동일지, Welldy
- **포지션**: 운동전문가를 위한 AI 업무 도구 → 웰니스 플랫폼
- **핵심 가치**: 기록·피드백을 "대신 만들어주는" AI
- **1순위 타겟**: 헬스 PT 트레이너 (1회 이상 수업 진행한 회원 담당)

---

## 핵심 문제

트레이너의 실력이 기록·소통의 병목 때문에 회원에게 전달되지 못함.

```
시간 없음 → 기록 대충/안 함 → 다음 수업 때 기억 안 남
→ 체계적 관리 불가 → 회원이 성장 못 느낌 → 재등록률 하락
```

기존 서비스는 "정리해주는 도구"이지, "대신 만들어주는 도구"가 아님.
드롭다운과 필드를 채우는 방식은 "또 다른 일거리"가 됨.

---

## 설계 원칙

1. **입력 부담 제로** — 자연어로 쏟아 붓기. 드롭다운/필드 채우기 없음
2. **회원 앱 설치 불필요** — 카톡/웹링크로 결과 전달
3. **비정형 데이터 수용** — 통증, 컨디션, 메모를 자연어 그대로
4. **일을 줄여주는 도구** — "또 다른 일거리"가 아닐 것
5. **기존 도구 보완** — 센터 CRM, 카톡 등을 대체하지 않음

---

## AI 설계 방향

### 챗봇이 아니다

Welldy의 AI는 ChatGPT 같은 채팅 인터페이스가 아님.
트레이너가 원하는 건 AI와 대화하는 게 아니라 **일을 빨리 끝내는 것**.

### AI-Powered Features

AI는 **버튼 뒤에 숨어있는 엔진**. 사용자는 AI를 의식하지 않음.
노션 AI, 캔바 AI, 그래머리처럼 특정 기능에 AI가 녹아들어 있는 방식.

### 핵심 패턴

모든 AI 기능은 같은 패턴을 따름:

```
입력 (자유롭게) → AI 생성 (버튼 한 번) → 사람이 확인/수정 → 전달
```

- **입력은 자유롭게**: 비정형 자연어, 축약어, 메모 수준도 OK
- **결과물은 깔끔하게**: 구조화된 포맷, 전문적이면서 읽기 쉬운 톤
- **사람이 최종 통제**: AI 초안 → 트레이너가 확인/수정 후 전달

기존 앱이 "템플릿 빈칸 채우기"라면, Welldy는 "자유 입력 → AI 구조화".
이 차이가 "또 다른 일거리" vs "일을 줄여주는 도구"를 가름.

---

## 1단계 (MVP): AI 운동일지

### 기능 정의

트레이너가 자연어로 수업 내용을 입력하면, AI가 구조화된 운동일지를 생성한다.

### 사용자 시나리오

1. 수업 직후 트레이너가 홈 화면 바로가기 한 번 터치 → Welldy 입력 화면
2. 자연어로 수업 내용 입력 (축약어, 비정형 표현 모두 가능)
3. [생성] 버튼 → AI가 구조화된 운동일지를 생성
4. 트레이너가 내용 확인 후 필요시 수정
5. [복사] 버튼 → 카카오톡에 붙여넣기 → 회원에게 전달

### 입력 예시 (트레이너 자연어)

```
민수님 오늘 스쿼트 80kg 8회 4세트, 벤치프레스 60kg 10회 3세트,
랫풀다운 50kg 12회 3세트 했습니다.
마지막 스쿼트 세트 폼 좀 흐트러졌고, 오른쪽 어깨 약간 불편하다고 해서
다음에 오버헤드 프레스는 빼려고 합니다. 수면 부족으로 컨디션 안 좋았음.
```

### 출력 예시 (운동일지)

```
📋 운동일지
2026.02.09 | 민수님

🏋️ 오늘의 운동
종목          무게     횟수    세트
스쿼트        80kg    8회     4세트
벤치프레스     60kg    10회    3세트
랫풀다운      50kg    12회    3세트

📝 코치 노트
- 스쿼트 마지막 세트에서 폼이 다소 흐트러짐 → 다음 세션에서 집중 체크
- 오른쪽 어깨 불편감 있어 다음 시간 오버헤드 프레스 제외 예정
- 컨디션: 수면 부족으로 전반적 피로감

다음 세션에서는 어깨 상태 확인 후 상체 프로그램 조정하겠습니다.
수고하셨습니다! 🙌
```

### 핵심 설계

- 코치와 회원이 같은 문서를 본다 (별도 버전 없음)
- 둘 다 읽을 수 있는 톤 (전문적이면서 이해하기 쉬운)
- 비정형 데이터 (통증, 컨디션, 메모) 자연스럽게 포함
- 축약어 인식: "벤치 60 10 3" → 벤치프레스 60kg 10회 3세트
- 유저 테스트 후 톤/포맷 지속 보완

### UI 구조

```
[회원 이름 입력]  [날짜 (오늘 자동, 수정 가능)]
[큰 텍스트박스 — 자연어 자유 입력]
   placeholder: "민수님 오늘 스쿼트 80kg 8회 4세트..."
[생성하기 버튼]

--- 생성 후 표시 ---
[포맷팅된 운동일지]
[복사하기 버튼]  [다시 생성 버튼]
```

### 입력 설계

- 회원 이름 (필수) — text input
- 날짜 — 자동으로 오늘 날짜 (수정 가능) — date input
- 수업 내용 — 자연어 텍스트박스 (형식 제한 없음) — textarea

### 공유 설계 (단계별 진화)

| 단계 | 방식 | 필요 기술 |
|------|------|----------|
| MVP | 복사 버튼 → 카톡 붙여넣기 | 프론트엔드만 |
| 1.5단계 | 카톡 알림톡 + 웹 링크 | 백엔드 + DB + 카카오 비즈메시지 |
| 2단계 | 앱 저장 + 카톡 링크 공유 | 앱 개발 |

- MVP: 회원에게 앱/계정 요구 없음. 트레이너도 로그인 없이 사용 가능
- 1.5단계: 웹 링크로 정돈된 일지 제공. 앱 설치 없이 확인
- 2단계: 앱 설치한 회원은 히스토리 열람. 설치 유도하되 강제 아님

### MVP 기술 아키텍처

| 구성 | 기술 |
|------|------|
| 프론트엔드 | 순수 HTML/CSS/JS (프레임워크 없음) |
| 백엔드 | Cloudflare Pages Functions |
| AI | Claude API (Anthropic) |
| 배포 | GitHub → Cloudflare Pages (자동) |
| 환경변수 | ANTHROPIC_API_KEY (Cloudflare 대시보드에서 설정) |
| 폰트 | 시스템 폰트 (-apple-system, BlinkMacSystemFont, sans-serif) |
| 디자인 | Apple 미니멀 스타일 (기존 디자인 시스템 계승) |

### MVP에서 필요한 것

- 자연어 텍스트 입력 폼 (이름 + 날짜 + 내용)
- AI API 연동 (Claude API, 서버사이드 프록시)
- 결과 출력 화면 + 복사 버튼
- 모바일 최적화 (트레이너가 폰으로 사용)
- 홈 화면 바로가기 (PWA 또는 웹앱 아이콘)

### MVP에서 필요 없는 것

- 회원 계정/로그인
- 데이터베이스 (기록 저장)
- 음성 입력
- 회원 앱
- 카카오 알림톡 연동

## 제품 로드맵

### 1단계: AI 운동일지 (MVP)

자연어 → AI 운동일지 생성 → 복사 → 카톡 전달

### 2단계: AI 도구 확장 + 공유 고도화

**AI 도구 추가:**

| 도구 | 설명 | 패턴 |
|------|------|------|
| AI 회원 메시지 | 오늘 수업 기반 후기/격려 메시지 초안 | 일지 데이터 → AI 생성 → 수정 → 카톡 전달 |
| AI 식단 피드백 | 회원 식단 사진 기반 피드백 초안 | 사진 첨부 → AI 분석 → 수정 → 전달 |
| AI 프로그램 설계 | 목표/컨디션 기반 운동 프로그램 초안 | 조건 입력 → AI 생성 → 수정 → 적용 |

**공유 고도화:**
- 복사/붙여넣기 → 웹 링크 공유 (회원이 예쁜 웹페이지로 확인)
- 카카오 알림톡 연동

### 3단계: 데이터 축적 → 인사이트

데이터가 쌓이면서 가능해지는 기능:
- 회원별 히스토리 (성장 그래프, 무게/볼륨 변화 추적)
- 주간/월간 리포트 자동 생성
- 이탈 예측 (출석 패턴 기반 리스크 알림)
- 트레이너 대시보드

### 4단계: 회원관리 + 커뮤니티

AI 도구 사용이 정착되면 자연스럽게 확장:
- 가벼운 회원 관리 (AI 일지에서 자동 생성된 회원 목록)
- 스케줄/예약
- 트레이너 커뮤니티 (노하우 공유)
- 교육 콘텐츠, 자격증/채용 정보

### 5단계: B2C 확장 (웰니스 플랫폼)

- 회원이 직접 Welldy에서 운동 기록 확인
- 셀프 운동 사용자를 위한 AI 코칭
- 웰니스 커뮤니티

### 확장 전략 핵심

```
AI 도구 (일을 줄여줌)
  → 매일 쓰는 도구가 됨
    → 데이터가 쌓임
      → 인사이트 가능
        → 회원관리도 여기서
          → 플랫폼
```

CRM은 시작점이 아니라 도착점. AI 도구에서 출발해서 자연스럽게 플랫폼으로 확장.
기존 CRM이 엑셀에 AI를 얹은 거라면, Welldy는 AI에서 출발해서 관리 기능이 붙는 것.
