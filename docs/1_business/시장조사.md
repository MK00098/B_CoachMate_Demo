## 1. PT 트레이너의 하루 일과

### 근무 구조
- 대형 센터: 아침 6시 출근 ~ 저녁 8-11시 퇴근, 주 6일 근무
- 소형 스튜디오: 8-10시간이지만 오픈·마감·영업·청소까지 담당
- **분할 근무(Split Shift)**: 오전 피크(06~10시) / 오후 피크(18~23시) 사이 공백 → 하루 12시간 이상 구속

### 수업 횟수
- 소형 센터: 하루 3-4회 (주 10-20회)
- 대형 센터: 하루 8-12회 (월 150-200회 소화 사례도 있음)

### 시간대별 일과

| 시간 | 업무 |
|------|------|
| 06:00-09:00 | 새벽/오전 PT + 운동 영상 촬영 |
| 09:00-11:00 | 개인 운동, 오후 프로그램 구성 |
| 11:00-14:00 | 밀린 운동일지 작성, 식단 피드백, 회원 카톡 답장 |
| 14:00-17:00 | SNS 마케팅, 블로그, 프로그램 설계, 공부 |
| 17:00-22:00 | 피크타임 연속 PT (4-6타임), 신규 회원 응대 |
| 22:00-23:30 | 마감 청소, 매출 정리, 남은 기록 입력, 내일 준비 |

### 4가지 노동 유형
- **육체적 노동** — 코칭, 자세 교정, 시범
- **정신적 노동** — 프로그램 설계
- **감정 노동** — 회원 관리, 동기부여, 영업
- **디지털 노동** — SNS 마케팅, 콘텐츠 제작

## 2. 핵심 고충

### 2-1. 운동 기록의 딜레마

> "운동일지가 중요한 건 아는데, 제대로 쓰려면 시간이 너무 깨지고, 대충 쓰면 쓸모가 없다"

- 세션당 권장 기록 시간: 15분 (수업 전 5분 + 중 5분 + 후 5분)
- 하루 10세션이면 기록에만 **2.5시간** → 현실적으로 불가능
- **실시간 기록 불가**: 수업 중 앱 입력은 코칭 흐름을 끊음
- **중복 노동**: 종이에 메모 → 나중에 앱/엑셀에 옮겨적기
- **비정형 데이터**: "어깨 통증 약간 있음" 같은 정성적 정보를 정형 필드에 넣기 어려움
- **기록 파편화**: 종이/엑셀/카톡/앱에 흩어져 나중에 못 찾음

### 2-2. 회원 관리·소통

- **카톡/DM 과부하**: 업무와 사생활 완전히 혼재
- **소통 채널 파편화**: 카톡, 네이버 밴드, 인스타 DM, 노션, 전화, 문자 등 분산
- **후속 관리 압박**: 수업 외 무료 감정노동 (식단 피드백, 안부, 동기부여)
- **개인화 한계**: 20-30명에게 매번 다른 맞춤 프로그램 → 물리적 불가 → 복붙식 프로그램

### 2-3. 구조적 문제
- 저녁·주말 없는 삶
- 불투명한 인센티브 (센터 70-80% : 트레이너 20-30%)
- 센터에 구속되나 수업 없는 시간은 무급에 가까움

### 핵심 악순환

```
시간 없음 → 기록 대충/안 함 → 다음 수업 때 기억 안 남
    → 체계적 관리 불가 → 회원이 성장 못 느낌 → 재등록률 하락
```

## 4. 운동일지 공유 방식 현황

> 출처: Gemini Pro + Perplexity Pro + Gemini Deep Research (2026.02)

### 4-1. 방식별 비교

| 방식 | 점유율 (추정) | 트레이너 워크플로우 | 회원 수신 형태 | 핵심 장점 | 핵심 단점 |
|------|-------------|-------------------|-------------|----------|----------|
| **카카오톡 직접** | 70%+ (압도적) | 메모 → 정리 → 카톡 전송 | 텍스트/사진/영상 | 설치 불필요, 열람률 최고 | 기록 휘발, 포맷팅 노동, 공사 혼재 |
| **전용 앱** (포인티, 헬린캠프, 플릭 등) | 10~30% | 앱에서 운동 선택 → 입력 → 레포트 발행 | 앱 푸시/알림톡+링크 | 전문적 이미지, 데이터 시각화 | 회원 앱 설치 저항, 이중 입력, 유료 |
| **네이버 밴드** | 5% 미만 | 비공개 밴드 → 게시글 작성 | 밴드 피드 | 아카이빙 용이, 그룹 소통 | 또 다른 앱 설치 필요 |
| **노션/구글시트** | 극소수 | 템플릿 세팅 → 실시간 입력 → 링크 공유 | 웹 링크 | 완벽한 커스터마이징 | 모바일 입력 불편, 학습 장벽 |
| **종이 일지** | 15% | 수업 중 수기 → 사진 찍어 카톡 전송 | 현장 열람/사진 | 수업 흐름 끊김 없음 | 분실, 데이터화 불가 |
| **센터 CRM** (온핏 등) | 센터 단위 | 출결/횟수 입력 → 회원앱에서 확인 | 앱/알림 | 운영과 통합 | 코칭 레벨 기록 미지원 |

### 4-2. 핵심 발견

**카톡이 왕인 이유:**
- 회원 100%가 이미 사용 중 — 설치/가입 제로
- 알림 확인율 최고 — 열람 보장
- 텍스트/사진/영상/링크 모두 전송 가능

**전용 앱 → 카톡 회귀 패턴:**
1. 회원이 앱 설치를 귀찮아함 → "앱 깔아주세요"가 스트레스
2. 앱에 기록했는데 회원이 안 봄 → 캡처해서 카톡으로 또 보냄 (이중 노동)
3. 앱 오류/점검 시 기록 날아감 → 신뢰 상실 → 카톡으로 회귀
4. 센터 CRM과 중복 → "어디에 뭘 적어야 할지" 혼란

**트레이너 시간 소요 비중:**
- 기록 작성 (60%) — 가장 큰 병목. 회원당 10분+, 하루 10세션이면 기록에만 100분+
- 공유/전달 (20%) — 도구가 잘 잡히면 자동화 가능
- 피드백 대응 (20%) — 시간보다 멘탈 소모가 큼

**최적 공유 방식 — "링크 기반 하이브리드":**
- 포인티, 플릭이 이미 이 방향: 트레이너는 앱/웹에서 기록 → 회원은 카톡 알림톡 + 웹 링크로 확인
- 앱 설치 없이도 레포트 열람 가능 → 진입 장벽 최소
- "앱 깔기 귀찮지만, 카톡으로 링크 오면 눌러본다"는 회원 행동과 일치

### 4-3. 트레이너/회원 실제 목소리

> "수업 10개 하고 나면 밤에 일지 보내다가 잠듭니다. 복붙하다가 회원 이름 틀려서 식겁한 적도 있고요."

> "회원님들이 레포트 받는 맛에 운동하신대요. 재등록을 먼저 이야기하시는 경우가 많아졌습니다." (포인티 사용 트레이너)

> "쌤이 보내주는 장문의 카톡이 고맙긴 한데, 한 달 전 기록이랑 비교하고 싶어도 대화창을 한참 올려야 해서 포기하게 돼요." (회원)

> "한 달에 3만 원씩 내는데 서버가 터져서 수업 기록이 날아간 뒤로는 그냥 카톡 보냅니다." (앱 회귀 트레이너)

### 4-4. Welldy 시사점

| 시장 현실 | Welldy 대응 |
|----------|------------|
| 카톡이 최종 전달 채널 | 카톡으로 복사/붙여넣기 + 이후 링크 공유 |
| 회원 앱 설치 = 사망 | 회원에게 앱 설치 강요 없음 |
| 기록 작성이 최대 병목 | 자연어 입력 → AI가 구조화 |
| 포맷팅이 감정 노동 | AI가 "예쁜 말 + 피드백"까지 생성 |
| "있어빌리티" 니즈 | 카톡 텍스트보다 정돈된, 앱보다 부드러운 톤 |
| 드롭다운 = 불편 | 자유 입력 ("벤치 60 10"도 인식) |

## 5. 미해결 Gap

### 5-1. 기존 서비스가 못 푸는 것

1. **입력 부담**: "정리해주는 도구"는 많지만, "대신 만들어주는 도구"는 없음
2. **데이터 파편화**: 센터 CRM / 개인 도구 / 카톡 / 회원 앱에 분산
3. **코칭 인사이트 부재**: 어떤 회원이 이탈할지, 어떤 패턴이 효과적인지 분석 없음
4. **감정 노동 경감 불가**: 10명+ 회원에게 각각 다른 톤으로 피드백 → 번아웃

### 5-2. AI 개입 기회

| 영역 | 기존 | AI 적용 후 |
|------|------|-----------|
| 운동 기록 | 직접 타이핑/메모 | 음성·자연어로 말하면 자동 구조화 |
| 회원 리포트 | 수동 작성 또는 없음 | 기록 기반 자동 생성 |
| 프로그램 설계 | 0에서 수작업 | AI 초안 → 트레이너 수정 |
| 회원 메시지 | 일일이 카톡 작성 | AI 초안 → 트레이너 확인 후 발송 |
| 이탈 예측 | 감에 의존 | 데이터 기반 리스크 스코어링 |
| 영상 활용 | 찍고 카톡에 보내고 끝 | 동작 인식, 자세 분석, 변화 추적 |

### 5-3. Welldy 포지셔닝

```
기존 서비스:  센터 CRM (운영) ←→ PT 기록 앱 (정리)
                                    ↑
Welldy:                 AI 코칭 코파일럿 (생성)
                   "기록·피드백을 대신 만들어주는 레이어"
```

기존 도구를 대체하는 것이 아니라, 그 위에 올라가는 **AI 레이어**.
